# Web Application Infrastructure (Варіант №23)

Даний проєкт реалізує відмовостійку та безпечну інфраструктуру для веб-застосунку на базі Node.js та PostgreSQL.

## Архітектура системи
Взаємодія компонентів побудована за наступною схемою:
1. **Nginx (Reverse Proxy)**: Приймає зовнішні HTTP-запити на порту 80 і перенаправляє їх на локальний сокет.
2. **Systemd Socket (Socket Activation)**: Прослуховує порт 8000. При надходженні першого запиту автоматично активує сервіс застосунку.
3. **Node.js App**: Веб-сервер на базі Express, що працює від імені ізольованого користувача `app`.
4. **PostgreSQL**: Система керування базами даних, що зберігає інвентарну таблицю `inventory`.

## Безпека та ролі
* **app**: Системний користувач без права входу в систему, від імені якого працює код застосунку.
* **operator**: Користувач із обмеженими правами `sudo` для керування сервісами та Nginx.
* **student**: Адміністратор системи з повним доступом через `sudo`.
